# 지식 그래프 재구성 — 로컬 임베딩 기반 자동 의미 연결 + EdgeInspector

작성일: 2026-05-26 · 대상: AI 업무도우미 지식 위키 그래프 (`/knowledge/graph`)

## 1. 목표 (1줄)

**카파시 LLM Wiki 구조 위에 로컬 임베딩(ChromaDB+ONNX)으로 자동 의미 엣지를 추가하고, 엣지 클릭으로 두 페이지를 한 화면에서 비교·검증·승격할 수 있는 운영 도구로 그래프를 재구성한다.**

## 3. Phase 1 — Backend

### 3.1 신규 모듈 `backend/app/services/knowledge_graph_builder.py`

```python
def build_graph(
    *,
    service: str | None = None,
    page_type: str | None = None,
    category: str | None = None,
    implicit_threshold: float = 0.75,
    implicit_max_per_page: int = 5,
) -> GraphResponse:
    # 1. 페이지 메타 + ChromaDB 임베딩 일괄 로딩 (page 단위 dedup)
    # 2. cosine similarity matrix
    # 3. explicit edges (knowledge_link_parser)
    # 4. implicit edges (cosine > threshold, top N per page, excluding explicit)
    # 5. networkx graph build → Louvain community
    # 6. degree centrality → godScore
    # 7. TF-IDF community labels
    # 8. Response dict 구성
```

### 3.2 `/api/v1/knowledge/graph` 응답 확장

```json
{
  "nodes": [{
    "id", "title", "pageType", "category", "service",
    "linksCount", "backlinksCount",
    "community": int,       // 신규
    "godScore": float,      // 신규 (degree centrality 정규화 0~1)
    "degree": int           // 신규
  }],
  "edges": [{
    "from", "to", "isBroken", "crossService",
    "kind": "explicit" | "implicit",   // 신규
    "weight": float,                    // 신규 (explicit=1.0, implicit=similarity)
    "similarity": float | null          // 신규 (implicit 만 값)
  }],
  "communities": [{                     // 신규
    "id": int, "label": str, "size": int, "color": str
  }]
}
```

### 3.3 신규 엔드포인트 `GET /api/v1/knowledge/edge`

쿼리: `?from={id}&to={id}`

응답:
```json
{
  "from": { ...KnowledgeDoc 전체... },
  "to": { ...KnowledgeDoc 전체... },
  "edge": {
    "kind": "explicit|implicit",
    "weight": float,
    "similarity": float | null,
    "isBroken": bool,
    "crossService": bool
  }
}
```

### 3.4 신규 엔드포인트 `POST /api/v1/knowledge/edge/promote`

implicit → explicit 승격:
```json
body: { "from": "id", "to": "id", "anchorText": "참고" }
```
- `from` 페이지 본문 끝에 `[[to-id]]` 추가
- 자동 PUT (version+1, changelog 자동, links 재파싱)
- 응답: `{ "from": "...", "to": "...", "newVersion": 4, "linkAdded": true }`

## 4. Phase 2 — Frontend

### 4.1 `KnowledgeGraphPage.tsx` 변경

- 엣지 클릭 핸들러: `onEdgeClick(from, to)` → `setActiveEdge({from, to})`
- 엣지 시각:
  - explicit: cyan 실선 (`#06b6d4`, 2px)
  - implicit: 회색 점선 (`#94a3b8`, 1.5px, dasharray "4,3", 굵기 = weight*3)
  - broken: 빨강 점선 (기존)
  - cross-service: 보라 (기존)
- 노드 색상: `community` 기준 7~10색 팔레트
- 노드 테두리: `service` 기준 (codeeyes=cyan, unknown=slate)
- 범례 갱신: kind + community + cross-service + broken

### 4.2 신규 컴포넌트 `EdgeInspectorPanel.tsx`

- 우측 슬라이드 드로어 (width ~70vw, height 100%)
- 상단: 닫기 + 관계 메타 배지 (kind / weight / similarity)
- 본문 3-col grid:
  - 좌 (40%): `MarkdownView` 페이지 A 본문 + 헤더 (제목·badge들·version)
  - 중 (20%): 액션 버튼 세로 정렬
    - "↗ A 페이지로 이동"
    - "↗ B 페이지로 이동"
    - **"⇄ implicit → explicit 승격"** (implicit 엣지에서만 활성, 모달 confirm + anchor text input)
  - 우 (40%): `MarkdownView` 페이지 B 본문 + 헤더
- API: `getEdge(from, to)` + `promoteEdge(from, to, anchorText)`

### 4.3 `api.ts` 클라이언트 추가
- `getEdge({ from, to })`
- `promoteEdge({ from, to, anchorText })`
