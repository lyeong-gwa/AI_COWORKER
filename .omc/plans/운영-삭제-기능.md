# 운영자 삭제 기능 — 워크플로 cascade 삭제 + 인스턴스DB records 관리

작성일: 2026-05-27 · 대상: AI 업무도우미

## 1. 목표 (1줄)

**운영자가 SSH/파일시스템 우회 없이 웹 UI 에서 워크플로(+실행이력 cascade) 와 인스턴스DB records 를 안전하게 삭제할 수 있게 한다.**

## 2. 확정 결정 (사용자 요청)

| # | 결정 |
|---|---|
| D1 | 워크플로 삭제 시 관련 **모든 실행이력·창고·노드결과** cascade 삭제 (쓰레기데이터 0) |
| D2 | 인스턴스DB record 단건 + 다중 삭제 지원 |
| D3 | 웹 UI 에 삭제 진입점 제공 (정책 갱신 필요) |
| D4 | 삭제 전 confirm 필수 (영향 카운트 명시) |

## 3. 산출

### 3.1 Backend — Phase A

**(A-1)** `DELETE /api/v1/workflows/{id}` cascade 완전성 검증·보강
- 모델 `Workflow` 가 이미 `cascade="all, delete-orphan"` 3건 보유 (workflow_nodes, workflow_connections, executions). 확인 후 누락 시 보강:
  - workflow_executions
  - workflow_executions → node_results (있다면)
  - **warehouse_entries** (workflow 단위 또는 execution 단위 외래키)
  - workflow_nodes / workflow_connections
- 삭제 전 `GET /workflows/{id}/delete-preview` 신규 → `{workflowId, instanceCount, warehouseEntryCount, willDelete: true}` 응답
- 실제 DELETE 시 트랜잭션, 실패 시 롤백

**(A-2)** `DELETE /api/v1/instance-dbs/{idb}/records/{rid}` 신규 단건 삭제
- 응답: `{deleted: true, recordId}`
- 미존재 → 404

**(A-3)** `POST /api/v1/instance-dbs/{idb}/records/delete` 신규 bulk 삭제
- body: `{recordIds?: string[], filter?: dict}` — 둘 중 하나 필수
- filter: data 필드 키 기반 (예: `{board_id: 2778}`)
- 응답: `{deletedCount, deletedIds}`
- 안전장치: filter 없이 recordIds 비어있으면 422

**(A-4)** 신규 테스트 `tests/test_workflow_cascade_delete.py` + `tests/test_instance_db_record_delete.py`
- 워크플로 cascade: 인스턴스 2건·창고 N건 있는 wf 삭제 → 모두 0 검증
- record 단건 삭제 + 미존재 404
- bulk 삭제 (filter, recordIds 양쪽) + 안전 가드

기존 242 회귀 0.

### 3.2 Frontend — Phase B

**(B-1)** `WorkflowListPage.tsx`
- 카드 우상단 또는 메뉴(⋯) 에 🗑 버튼 추가
- 클릭 시 **WorkflowDeleteConfirmModal** (신규):
  - 워크플로 이름·id·노드 수
  - **함께 삭제될 데이터**: 인스턴스 N건, 창고 entry M건 (`/delete-preview` 호출로 미리 카운트)
  - 입력 confirm — 워크플로 id 또는 이름 타이핑해야 활성
  - 삭제 → `DELETE /workflows/{id}` → 성공 토스트 + 목록 refetch

**(B-2)** `InstanceDBViewerPage.tsx`
- records 테이블 행마다 🗑 단건 삭제 버튼 + confirm 미니 모달
- (선택) 행 체크박스 + 상단 "선택 N건 삭제" 버튼
- 삭제 → `DELETE /records/{rid}` 또는 `POST /records/delete` → 목록 refetch

**(B-3)** 화면 안내문 갱신
- "생성·수정·삭제는 CLI로만 가능합니다" → "생성·수정은 CLI로, **삭제는 웹 UI 에서도 가능합니다**"
- 두 페이지 모두

### 3.3 정책 갱신 — Phase C

- `AI 업무도우미/CLAUDE.md` §5.1·§6.10 갱신:
  - 워크플로 / 인스턴스DB **삭제**는 웹 UI 허용
