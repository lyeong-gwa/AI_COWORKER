# 시스템 아키텍처

AI 업무도우미의 전체 기술 아키텍처와 데이터 흐름을 설명합니다.

---

## 전체 구조

```
┌─────────────────────────────────────────────────────┐
│                    Frontend                          │
│  React 19 + TypeScript + Vite 7 + Tailwind CSS      │
│  @xyflow/react (워크플로우 캔버스)                     │
│  react-markdown (마크다운 렌더링)                      │
├─────────────────────────────────────────────────────┤
│                   REST API / SSE                     │
├─────────────────────────────────────────────────────┤
│                    Backend                           │
│  FastAPI + SQLAlchemy (비동기)                        │
├──────────────┬──────────────┬────────────────────────┤
│   SQLite DB  │  ChromaDB    │   LLM API (GPT-4o)    │
│  (관계형 데이터) │ (벡터 DB)    │  (AI 추론)             │
└──────────────┴──────────────┴────────────────────────┘
```

## Frontend 기술 스택

| 기술 | 버전 | 용도 |
|------|------|------|
| React | 19 | UI 프레임워크 |
| TypeScript | 5.x | 정적 타입 |
| Vite | 7 | 빌드 도구 및 개발 서버 |
| Tailwind CSS | 3.x | 유틸리티 기반 CSS (다크 테마) |
| @xyflow/react | - | 워크플로우 캔버스 (노드 그래프) |
| React Router | v6 | 클라이언트 라우팅 |
| react-markdown | - | 마크다운 렌더링 |

### 프론트엔드 설계 패턴

- **API-first with mock fallback**: 모든 페이지가 백엔드 API를 우선 호출하고, 실패 시 로컬 Mock 데이터로 대체하여 오프라인에서도 동작
- **Code splitting**: React.lazy() + Suspense로 페이지별 청크 분리, 초기 로드 최적화
- **ChatContext**: 페이지별 AI 어시스턴트 컨텍스트 연동, AI 액션 후 자동 리프레시

## Backend 기술 스택

| 기술 | 용도 |
|------|------|
| FastAPI | 비동기 REST API 서버 |
| SQLAlchemy (async) | ORM, 비동기 데이터베이스 접근 |
| SQLite | 경량 관계형 데이터 저장 |
| ChromaDB | 벡터 데이터베이스 (문서 임베딩 저장 및 유사도 검색) |
| ONNX Runtime | 로컬 임베딩 모델 실행 (외부 API 불필요) |
| SSE (Server-Sent Events) | 워크플로우 실행 실시간 스트리밍 |

### 백엔드 API 설계

- **RESTful 엔드포인트**: CRUD 전 엔드포인트 지원
- **Pagination**: `skip`/`limit` 파라미터로 페이지네이션
- **Search**: `q=keyword` 파라미터로 검색
- **camelCase 응답**: Pydantic 스키마에서 `serialization_alias`로 프론트엔드 컨벤션에 맞게 응답

## 데이터 흐름

### 1. 지식 기반 AI 추론 (RAG)

```
사용자 질문
    ↓
ONNX 임베딩 모델로 질문 벡터화
    ↓
ChromaDB에서 유사도 검색 (상위 N건)
    ↓
관련 지식 문서 + 원본 질문을 LLM에 전달
    ↓
LLM이 지식 기반 답변 생성
    ↓
답변 + 참조 문서(유사도 점수) 반환
```

### 2. 워크플로우 실행

```
사용자가 "실행" 클릭
    ↓
Backend에서 워크플로우 그래프 순회
    ↓
각 AI 노드별:
  - 바인딩된 도구 실행
  - LLM API 호출
  - 결과를 다음 노드에 전달
    ↓
SSE로 실행 진행 상황 실시간 스트리밍
    ↓
실행 완료 후 이력 저장
```

### 3. AI 태스크 자동 생성

```
사용자가 AI 채팅에서 태스크 생성 요청
    ↓
AI가 지식 베이스에서 관련 문서 검색
    ↓
태스크 제목, 설명, 우선순위, 태그 자동 설정
    ↓
태스크 생성 + 활동 이력 기록
    ↓
관련 지식 문서를 참조 자료로 연결
    ↓
지식 기반 답변 댓글 자동 생성
```

- **API 에러 상세화**: 백엔드 에러 메시지 추출 후 토스트로 표시
- **API 연결 상태 모니터링**: 사이드바에서 백엔드 연결 상태를 실시간 표시
